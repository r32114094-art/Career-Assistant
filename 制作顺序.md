# GenAI Career Assistant 优化与重构执行顺序 (Zero-Waste Roadmap)

为了保证后续引入前端页面和高级高级特性时**无代码返工 (Zero-Waste)**，项目必须严格按照由内向外、由底层逻辑到顶层展现的顺序进行重构与开发。

---

## 阶段零：前置必要改造 —— 状态机与持久化记忆剥离 (核心骨架大换血)
> **目标**：彻底丢掉局部的硬编码循环，让系统变成纯粹的“状态机”，这是后续所有进阶操作的绝对基石。

- **1. 重构 State (`state.py`)**：引入统一的 `messages` 列表或针对不同场景的记忆分区。
- **2. 拆除内部阻塞 (`learning_agent.py` / `interview_agent.py`等)**：强制删除所有 Agent 节点内部的 `while True:` 和 `input()`。所有节点改为接收当前状态 -> 触发一次 LLM -> 返回新状态即结束。
- **3. 中心化路由 (Supervisor)**：升级 `categorize` 节点，使其能够在每次流转前重新评估用户的全局最新意图，实现跨频道无缝跳跃。
- **4. 引入持久化机制 (Checkpointer)**：在 `workflow.py` 编译大图时挂载 `MemorySaver`，使图支持基于 `thread_id` (会话ID) 的跨轮次记忆。
- **5. 顶层驱动 (`main.py`)**：将唯一的命令行对话 `input()` 放在这里，通过 `app.invoke` 持续推给大图。

---

## 阶段一：高阶 Agentic 特性开发 (秀肌肉阶段)
> **目标**：趁着还在本地命令行方便调试，快速开发出简历上最亮眼的、传统 Agent 无法实现的复杂策略。

- **1. 错误边界与自我反思重试 (Self-Reflection)**：
  - 在诸如职位搜索 `job_search` 节点，如果搜索结果解析失败或为空，不直接报错给用户，而是让节点触发内置策略重试其他关键词（无需人类干预）。
- **2. 实施人机协同 (Human-in-the-loop)**：
  - 在发送面试题或生成最终 PDF 简历前，利用大图的 `interrupt_before` 功能将图执行挂起。
  - 在终端提示框中询问用户是否满意（审核确认），满意则继续，不满意则原路退回调整。此逻辑不仅能跑在终端，未来可无缝平移到网页前端的“弹框确认”操作中。

---

## 阶段二：大模型量化监控体系接入 (评估与背书)
> **目标**：在系统逻辑完全冻结后，建立数据化评估标准，为面试简历积累绝对的“量化成就”。

- **1. 接入平台级监控**：注册并配置 LangSmith API Key，开启全链路 Trace 监控，获得图运转时的耗时、Token 消耗以及工具调用追踪图。
- **2. 建立评估基准集 (Golden Dataset)**：编写 Python 脚本，自动化向重构后的 Agent 灌入 50-100 道真实职业测试题。
- **3. LLM-as-a-judge 评估**：提取系统日志，针对“意图路由准确率”、“搜索工具使用成功率”、“上下文无污染率”制作量化指标集。

---

## 阶段三：后端接口化与 Web 全栈打通 (最终产品化)
> **目标**：为这个强大的内部大脑套上商业级产品的外衣，实现大模型应用全栈交付。

- **1. 后端接口封装 (FastAPI)**：
  - 弃用单纯的 `main.py` CLI 入口。
  - 用 FastAPI 编写 RESTful 接口（如 `@app.post("/chat")`），它的底层直接调用我们第一阶段完成的无状态图 `app.invoke()`。对前端返回结构化的 JSON 或流式数据 (SSE)。
- **2. 状态映射与恢复对接**：
  - 让前端发来的请求携带 `thread_id`，利用底层的 Graph Checkpointer 完成对断点、中断、追问等操作的无感延续。
- **3. 现代前端应用搭建 (Next.js / React)**：
  - 基于 Next.js（或更简便的 Streamlit）结合 Vercel 进行快速前端项目初始化。
  - 编写聊天 UI 组件、Markdown 渲染面板以及打断状态专用的“Approval/Reject (审核/拒绝)”控制按钮交互。
